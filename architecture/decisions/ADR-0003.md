# ADR-0003: Entity ID Schema

**Status:** Accepted | **Date:** 2026-07-17 | **Author:** Principal Architect

## Context
Every entity in Storynaram needs a unique, human-readable identifier that encodes entity type and supports sorting.

## Decision
Adopt `{entity_type_prefix}_{zero_padded_sequence}` format:
- `hero_000001`, `book_000042`, `city_001337`
- Prefixes: 2-5 lowercase characters describing entity type
- Sequence: minimum 6 digits, zero-padded
- Sequence is globally unique per prefix

## Alternatives Considered
1. UUID — Unreadable, not sortable by creation, no entity type encoding
2. Incremental integers — No entity type information, collision risk across types
3. Semantic IDs (e.g., `hero-aldric`) — Not globally unique, naming conflicts

## Rationale
- Human-readable and recognizable at a glance
- Self-documenting (prefix tells you what it is)
- Sortable lexicographically by creation order
- Consistent across all 86 entity types

## Consequences
- ID validation regex needed per prefix
- Prefix registry must be maintained
- Sequential IDs reveal entity count (acceptable for single-user)
- Zero-padding ensures correct sort order

## Future Impact
- Phase 2 Schema: ID format must be encoded in JSON Schema
- Phase 3 Validation: ID format validator needed
- Phase 9 Database: IDs become primary keys
