# ADR-0005: AI Knowledge Architecture

**Status:** Accepted | **Date:** 2026-07-17 | **Author:** AI Platform Architect

## Context
The AI system needs to understand story context, retrieve relevant knowledge, maintain canon integrity, and provide intelligent assistance — all without coupling to specific AI models.

## Decision
Build a modular AI architecture with:
- Model-agnostic pipeline (supports OpenAI, Claude, Gemini, local models)
- Knowledge graph for entity relationship traversal
- Context Builder with priority-based context optimization
- Canon Engine with immutable records and conflict detection
- Memory system (short-term, long-term, working)
- Validation pipeline as mandatory stage

## Alternatives Considered
1. Monolithic AI system — Tight coupling, hard to evolve
2. Model-specific implementation — Vendor lock-in
3. No AI architecture — AI is ad-hoc; rejected — causes inconsistency

## Rationale
- AI model landscape changes rapidly; abstraction is essential
- Story integrity requires canon protection independent of AI model
- Context optimization is critical for different model context windows
- Modular design enables incremental implementation

## Consequences
- 23 AI module directories (comprehensive)
- 42 files for AI architecture documentation
- Context Builder must handle varying context windows (4K-1M tokens)
- Canon Engine requires human-in-the-loop for locked entities

## Future Impact
- Phase 4 Prompts: Prompt library for each agent
- Phase 5 Memory: Memory system implementation
- Phase 8 AI Deepening: Multi-model orchestration
