# Tag Standard

## Purpose
Defines the tag taxonomy and usage conventions for all entities in Storynaram. Tags enable cross-domain filtering, search, and categorization.

## Tag Format
- **Lowercase** — all tags are lowercase: `protagonist`, `fantasy`, `draft`
- **No spaces** — use hyphens for multi-word: `point-of-view`, `high-fantasy`
- **Single words preferred** — `magic` over `magical-abilities`
- **Maximum length** — 30 characters per tag

## Tag Usage Rules
1. **Multiple tags allowed** — no limit on tag count per entity
2. **Relevant only** — tags must accurately describe the entity
3. **No redundancy** — don't tag `human` if the entity type is already `character`
4. **Hierarchical** — use general and specific tags: `fantasy`, `high-fantasy`
5. **Consistent** — use established tags before creating new ones
6. **New tags** — must be added to this document if used more than 3 times

## Tag Location in JSON
Tags are stored in the `tags` array at the top level of every entity:

```json
{
  "id": "hero_000001",
  "tags": ["protagonist", "human", "mage", "draft"]
}
```
