# Archive Directory

## Purpose
The historical archive system. Deprecated data, old versions, superseded content, and historical records preserved for reference.

## Responsibility
Manages the project's historical record â€” content that is no longer current but must be preserved for reference, legal, or historical purposes.

## Naming Convention
- **Subdirectories**: By date or by domain (e.g., 2026/, old_world/)
- **Files**: Original names with archive markers if needed
- **Structure**: Mirrors source structure where possible

## Contents
- Deprecated entity definitions
- Old versions of world data
- Superseded character definitions
- Previous timeline versions
- Old plot outlines
- Discarded story concepts
- Previous config versions
- Old schemas and templates
- Historical backups (pre-migration)

## Future Expansion
- Archive indexing and search
- Archive restoration tools
- Archive-to-current diff tools
- Archive retention policy engine
- Automated archiving rules
- Archive compression and storage optimization

## Relationships
- **Backups/** backups are separate from archives (backups = restore, archives = reference)
- **All directories** any directory may have archived content
- **Logs/** archiving operations logged
- **Memory/decisions** decision records explain why content was archived
- **Config/** archiving policies and rules
